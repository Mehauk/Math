Functions

Functions can also be represented in code:

def some_polynomial_function(x: float) -> float:
   return 3*x*x + 7*x + 14

A limitation due to the discrete nature of digital computers is that any function also becomes discrete. Although f(x)=2x2+7x+14{f(x) = 2x^2 + 7x + 14}f(x)=2x2+7x+14 is a continuous function both the input to the function xxx and the output are floats, and as such they have a limited range and precision.

Here is another example that more clearly demonstrates this:

fn another_polynomial_function(x: i32) -> i32 {
   (0.5 as i32)*x
}

This translates to f(x)=12x{f(x) = {1\over2}x}f(x)=21​x and the domain is as follows:

D:{x∣x∈R}{D: \{x| x \in \mathbb{R}\}} D:{x∣x∈R}

However The domain of an i32 is:

D:{x∣−231≤x<231∣x∈I}{D: \{x| -2^{31} \le x <2^{31} | x \in \mathbb{I}\}} D:{x∣−231≤x<231∣x∈I}

Additionaly an input of x=1{x = 1}x=1 would return 0.50.50.5, but since an i32 cannot represent fractional values, it would return 000;
